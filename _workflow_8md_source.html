1 Workflow of ExaGeoStat
2 ======================
3 The Exascale Geo-Statistics project (ExaGeoStat) is a parallel high performance
4 unified framework for computational geostatistics on manycore systems. The
5 software project aims at optimizing the likelihood function for a given spatial data
6 in order to provide an efficient way for predicting missing observations in the context of
7 climate/weather forecasting applications. The framework has three main components:
8 
9 
10 1. Generate synthetic datasets in the case of testing mode. The synthetic dataset
11  includes 2D spatial locations and measurements vector. The user can enable testing
12  mode by using (--test) input. Furthermore, real datasets can be used with
13  ExaGeoStat, when passing input files names for both spatial locations and measurements.
14 
15 2. An iterative procedure is performed over the given data (i.e., synthetic or real dataset)
16  to estimate the parameter vector the maximize the likelihood function for the given
17  data. The average number of iterations depends on the lower and upper bounds of the
18  optimization algorithm given by the user. Three computation options are available here,
19  Exact, Diagonal Super Tile (DST), and Tile Low-Rank (TLR)
20 
21 3. The estimated parameter vector (i.e., variance, smoothness, and range) of the Matern matrix kernel can be used
22  to predict measurements on missing spatial locations. The ExaGeoStat's prediction stage
23  is able to predict the measurements from the MLE parameter vectors. This stage is also running
24  in two different modes: test and real. For testing, unobserved spatial locations are randomly chosen from
25  the original spatial locations and their measurements are marked as unknown. In this
26  case, ExaGeoStat is able to compute the Mean Square Error (MSE) by comparing observed
27  measurements with predicted measurements. For real cases, the user provides the program
28  with the new spatial locations and the program predict the measurements for those new locations.
